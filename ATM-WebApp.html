ATM WEBAPP

BRIEF

Creation of a new responsive webapp For atm Milano, including new Services with a strong focus on User Experience.

CONCEPT

The Project started with a deep analisys on ATM vending machines that begun with interviews, the creation of personas and also from their inputs the aim of the Project was to simplify the structure of the whole purchasing system. The last step has been evolving the project into a responsive webapp usign the features of desktop and mobile phones in particular ( such as camera and geolocalisation ) to offer a better service For People.

MAP TOOL

The map tool is always visible to prevent errors. It suggests the way for you to reach any place, speeding up the ticket purchase. The search bar helps increasing the efficiency of the service. It also give the possibility to save the ticket and use it when it is necessary. Finally the Login comes after the map tool to allow to use the webapp also just for quick infos, giving immediate feedback to the user.

TICKET PURCHASE

The template is fixed to focus the user and rassure him without scrolling. There are no differences between city areas and it's added the possibility to save the ticket for later usage.

URBAN TICKET SHORTCUT

Researches done in the Milan Area pointed out that the majority of people search for â‚¬1,50 ticket. For this reason it has been introduced a new function that, saving clicks, allows users to purchase the urban ticket very fast from the main menu of the app. Since this is a new feature and people have to get used to it, the app has a brief description next to the call to action to prevent any error.

TRAVEL CARD

This particular funtion represented a problem for mobile users that have to fill in a crowded form, to make it easier, exploiting the features of a common smartphone, it has been introduced the possibility to scan card's data with the camera; this is a way, together with inline validation, to prevent errors. Another problem occurred while choosing between a list of contracts that in the mobile was too crowded to tap the one that suites for the users' needs. For this reason it has been added a shortcut to recharge the last contract the user did (which is quite often the choice he selects), giving him a clear resume and the possibility to find a totem where he can activate the card.
